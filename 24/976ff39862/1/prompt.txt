## Implementation Changes

- Codacy 조회는 account API token을 환경변수로만 받아 실행한다. 토큰은 파일에 저장하지 않는다.
- Codacy source: official `listFiles` API returns `path`, `gradeLetter`, `totalIssues`, `complexity`, `coverage`, `duplication`: [Codacy docs](https://docs.codacy.com/codacy-api/examples/obtaining-code-quality-metrics-for-files/).
- 작업 흐름:
  - Codacy API 전체 pagination 수집.
  - 로컬 `coverage json` 결과와 path 기준 join.
  - `F -> E -> D` 순서에서 Python source + missing lines > 0인 파일만 선택.
  - 기존 테스트 패턴에 맞춰 behavior-focused unit test 추가.
  - 한 번에 위 후보 1-2개씩 처리해서 review 범위를 작게 유지.

## Assumptions

- 이번 범위는 “테스트 보강으로 coverage를 올릴 수 있는 Python source”다.
- Codacy grade는 coverage 외에 issues, complexity, duplication도 포함하므로, coverage가 100%인데 F인 파일은 이번 작업에서 제외하고 별도 refactor/duplication 작업으로 분리한다.
- 토큰이 대화에 노출됐으므로 작업 후 Codacy에서 rotate/revoke하는 것을 권장한다.
